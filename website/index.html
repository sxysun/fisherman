One product, three context homes

Choose where private context lives.

01

Local Only

Raw context stays on your laptop. Friend status still works through the encrypted relay.

02

Fisherman Cloud

Managed backend with TDX attestation, multi-tenant ingest, and explicit client approval.

03

Self-Hosted

Run the same backend on your own server and keep the official encrypted relay for interop.

Privacy model

Trust is explicit, not implied.

Local mode avoids raw upload. Cloud mode requires attestation before private context leaves the device. Friend status is encrypted per recipient, so the relay stores signed ciphertext rather than plaintext.

01Persistent user identity

Your key stays stable across installs and signs protocol traffic.

02Client-held Cloud data keys

New Cloud deploys need renewed approval before they can compute on history.

03Portable context homes

Export, import, and delete history when moving between Local, Cloud, and Self-Hosted.

Agent Access

Give tools scoped context, not your whole machine.

create

Generate an expiring deputy token with exact scopes.

register

Paste the setup block into a remote agent host.

query

The agent reads captures, transcripts, screenshots, or status only if granted.

revoke

Remove access without rotating your Fisherman identity.
